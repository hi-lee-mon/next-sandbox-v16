例えばserver-onlyを使わない関数があるとする。

これはただの関数なのでどこからでもimportできる。
どこで実行されるかの違いなのでどちらの関数なのかわからない。

大きな違いとしてはserver functionは機密情報にアクセスできる。

例えばSCでこれをimportするとしよう。

そのあと何らかの理由でSCからCCに変えたとき、server functionはCC二送信されてしまう。

しかし、nextの場合はNEXT_PUBLICのついていない環境変数を読み込んでもundefinedになるので一応問題はない

問題は他にもある。

- 変数の値 は守られても、コード自体（関数・URLなど） はバンドルに含まれるので、それは読める。
- 誤ってserver関数がclientにimportされてバンドルサイズが膨らむのを防ぐ。
- fsやpathのようなサーバーでしか使えない関数を使っているとruntimeでしか気づけ無い
- server専用なのかどうかわかりづらい

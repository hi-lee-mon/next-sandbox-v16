サーバーで実行される関数でserverコンポーネントとクライアントコンポーネントの両方から呼び出すことができる。

ポイントはクライアントコンポーネントから呼び出す場合のみでそこだけ理解しておけばOK.

クライアントコンポーネントから呼び出す場合は、use serverディレクティブをつける。
もしつけない場合は、普通の関数なので自明だけど**クライアント側で実行されるし、クライアントバンドルに含まれる。**

サーバーコンポーネントで呼び出す場合は、ディレクティブがなくても実行環境がサーバーなのだからサーバーで実行される。

## 3つの重要ポイント
1つ目は前述したようにuse server ディレクティブをつけることでその関数はサーバー側で実行されるのでクライアントバンドルから除外される。

2つ目のポイントが機密情報にアクセスできるということ。サーバー側なので機密情報に安全にアクセスできる。これによりクライアント側で機密情報を持つ必要がなくなるので安全。

3つ目はディレクティブをつけない場合、サーバー、クライアントの両方で使える関数になるということ。これはとても重要なポイント。なぜなら機密情報を扱っている場合、クライアント側はエラーになるから。

Nextjsは.envを読み込むときに環境変数にNEXT_PUBLICがついていない場合はクライアント側で読み込めない仕様となっている。これは安全なのでとても良い機能だが、importして実際に実行されるまでエラーになることに気づかない。クライアント側で環境変数を呼び出しても読み込めずundefinedが返るだけでコードをエラーにはしてくれない。ちょっと不便だけどそういう仕様。

エラーにならないのでbuildしてciが通る。したがって、リリースしたあとにエラーに気づく可能性がある。なのでサーバーで実行しないと確実にエラーになるものはserver-onlyというモジュールをimportすると良い。importすることでCCでimportされていたらbuildエラーにすることができる

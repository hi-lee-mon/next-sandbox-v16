fetchはSCでやって更新をCCでやる。SCとSPAのハイブリット（blogページのコメントだけccにしつつ、コメントのいいねボタンなどはこれすると良さそう？）
グローバルな状態の共有が面倒かも？updateTagとかNextの機能を使ったパージができないから。

CCからの同じrouteでの更新は
```ts
"use server";

import { refresh, updateTag } from "next/cache";
import { BLOGS_CACHE_TAG } from "../../blogs/_data/cache-tags";

export const updateBlogTag = async () => {
  updateTag(BLOGS_CACHE_TAG);
  refresh()
}
```
でいける。

このとき、CCは
```tsx
  const [isPending, startTransition] = useTransition();
  return (
    <form onSubmit={
      async (e) => {
        e.preventDefault();
        startTransition(async () => {
          await updateBlogTag();
        });
      }
    }>
      <Button type="submit" disabled={isPending}>
        {isPending ? "更新中..." : "ブログを最新に更新"}
      </Button>
    </form>
```
のようになる。

仮にSCがたくさんあると下記の動作になる。
1. submit
2. action発火 + isPending=true(RSCストリーミング+コミット完了までpending)
3. server function発火
4. キャッシュパージ
5. refresh
6. レスポンス
7. server function終了
8. 現在のページのすべてのSCが再実行
9. SCがストリーミング開始
10. ストリーミング完了
11. コミット
12. isPending解除

このようにストリーミングに時間がかかるとtransitionの完了に時間がかかる。

ただし、use cacheでキャッシュが残っているものはすぐに返却されるためisPendingの時間には関係しない

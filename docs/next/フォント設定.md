デフォルトだと以下のようになっている。
Arialはwindowsのデフォルト、Helveticaはmacのデフォルトで必ず入っているフォント。
これらが見つからない場合にsans-serifが適用されるので自分のフォントを使う場合はここを削除するor自分でbody直下を上書きして書く必要がある
```css
body {
  background: var(--background);
  color: var(--foreground);
  font-family: Arial, Helvetica, sans-serif;
}
```

nextの場合はフォントがbuiltinされるのでbodyのfont-familyを削除しつつ、classNameを適用させても良い
```tsx
    <html
      lang="jp"
      className={`${noteSansJp.className} h-full antialiased`}
    >
      <body className="font-sans min-h-full flex flex-col">{children}</body>
    </html>
```

もしtailwindcssを使いたいならvariableを設定したのち
```tsx
const noteSansJp = Noto_Sans_JP({
  weight: ["400", "700"],
  variable: "--font-noto-sans-jp",
})

className={`${noteSansJp.variable} h-full antialiased`}
```
globals.cssで上書きする
```css
@theme inline {
    --font-sans: var(--font-noto-sans-jp);
```
これで
```tsx
<p className="font-sans">
```
で適用できる

あと、weightを絞ることでバンドルサイズが軽くなる。

最後にユーザが慣れ親しんでいる環境が一番使いやすい（読みやすい）であろうと考えて
```css
font-family: system-ui, sans-serif;
```
とするのが良い。
このとき、左が読んでいってsystem-uiが使えなければsans-serifが適用されるという順番。
system-uiの場合はユーザ環境のフォントが適用される。
sans-serifはヒゲなしのフォントのことで、どのフォントになるかはブラウザ依存。

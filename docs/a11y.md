# フォームで抑えておくべきa11y属性---

## 必須の4属性（これだけは絶対に入れる）

### `aria-required`

```tsx
<input aria-required={true} />
// または HTML標準の
<input required />
```

`required` と `aria-required` は役割が微妙に違います。`required` はブラウザのネイティブバリデーションを発動させますが、`aria-required` はスクリーンリーダーへの宣言です。RHFを使っている場合はブラウザバリデーションをオフにするので、両方書くのが無難です。

```tsx
<input
  required
  aria-required={true}
/>
```

---

### `aria-invalid`

```tsx
<input aria-invalid={!!errors.email} />
```

エラーがある状態でスクリーンリーダーがフォーカスすると「メールアドレス、無効な入力」と読み上げます。`true` / `false` の切り替えだけでOKです。

---

### `aria-describedby`

```tsx
<input
  aria-describedby={
    [
      errors.email ? 'email-error' : null,
      'email-helper',  // ヘルパーテキストのid
    ]
      .filter(Boolean)
      .join(' ')
  }
/>

<p id="email-helper">登録済みのメールアドレスを入力してください</p>
<span id="email-error">...</span>
```

スペース区切りで複数のidを指定できます。ヘルパーテキストとエラー両方を紐付けるのが正しい使い方です。

---

### `role="alert"`

```tsx
{errors.email && (
  <span id="email-error" role="alert">
    {errors.email.message}
  </span>
)}
```

`role="alert"` は要素が**DOMに出現した瞬間**に読み上げます。エラーが消えたり出たりする実装と相性が良いです。

---

## 状況に応じた属性

### `aria-live` と `aria-atomic`

`role="alert"` より穏やかに読み上げたい場合に使います（フォームよりはチャットのメッセージ通知向き）。

```tsx
// alert = aria-live="assertive" + aria-atomic="true" の省略形なので
// フォームエラーは role="alert" だけで十分なことが多い

<div aria-live="polite" aria-atomic="true">
  {statusMessage}
</div>
```

### `aria-errormessage`

`aria-describedby` のエラー専用版ですが、**サポート状況がまだ不安定**なため、現状は `aria-describedby` で代用するのが安全です。

```tsx
// 将来的にはこちらが推奨になる可能性がある
<input
  aria-invalid={true}
  aria-errormessage="email-error"
  aria-describedby="email-error"  // 今は両方書く
/>
```

### `aria-disabled`

```tsx
// disabled だけだとフォーカスできず、スクリーンリーダーに読まれない場合がある
// 読み上げさせたい非活性ボタン（理由を伝えたい）には aria-disabled を使う

<button
  aria-disabled={isLoading}
  onClick={(e) => { if (isLoading) e.preventDefault() }}
>
  送信
</button>
```

## まとめ：RHFでの実装パターン

```tsx
function EmailField() {
  const { register, formState: { errors } } = useFormContext()
  const error = errors.email

  return (
    <div>
      <label htmlFor="email">
        メールアドレス
        <span aria-hidden="true">（必須）</span>
      </label>

      <input
        id="email"
        type="email"
        required
        aria-required={true}
        aria-invalid={!!error}
        aria-describedby={[
          'email-helper',
          error ? 'email-error' : undefined,
        ]
          .filter(Boolean)
          .join(' ')}
        {...register('email')}
      />

      <p id="email-helper">
        登録済みのメールアドレスを入力してください
      </p>

      {error && (
        <span id="email-error" role="alert">
          {error.message}
        </span>
      )}
    </div>
  )
}
```

`aria-hidden="true"` を `（必須）` に付けているのは、`aria-required` で既にスクリーンリーダーに必須であることが伝わっているため、「必須」と二重に読み上げないようにするためです。

# フォームのエラーメッセージ設計
## 設計の前提：ユーザーがフォームで迷う原因

1. **何を入力すればいいかわからない** → ラベル・プレースホルダーで解決
2. **入力ルールを知らない** → ヘルパーテキストで解決
3. **間違えたことに気づかない / 直し方がわからない** → エラーメッセージで解決

## 各要素の役割と設計原則

### ラベル（Label）
- **必須**。プレースホルダーで代替しない
- 簡潔に「何か」だけを伝える
- 必須項目には「（必須）」を付ける

### プレースホルダー（Placeholder）
- 入力形式の**例示**のみに使う
- 消えるので、ルール説明には使わない

### ヘルパーテキスト（Helper Text）
- 入力**前**に伝えるべき補足情報
- 「〜の形式で入力してください」はここ

### エラーメッセージ（Error Message）
- **何が悪いか** + **どう直すか** をセットで伝える
- 「正しく入力してください」は絶対NG（何が正しいのか不明）
- ユーザーを責めない文体にする

## エラーパターン別メッセージ設計

### メールアドレスフィールド

| タイミング | 状況 | メッセージ |
|---|---|---|
| 未入力でsubmit | 空欄 | `メールアドレスを入力してください` |
| blur時 or submit | 形式が不正 | `メールアドレスの形式で入力してください（例: name@example.com）` |
| submit後（API） | 存在しないアカウント | `メールアドレスまたはパスワードが正しくありません` ※1 |

### パスワードフィールド

| タイミング | 状況 | メッセージ |
|---|---|---|
| 未入力でsubmit | 空欄 | `パスワードを入力してください` |
| blur時 | 文字数不足 | `パスワードは8文字以上で入力してください` |
| submit後（API） | 認証失敗 | `メールアドレスまたはパスワードが正しくありません` ※1 |

> **※1 セキュリティ上の重要ポイント**
> 「パスワードが間違っています」と個別に伝えると、攻撃者にアカウントの存在を教えてしまいます。認証失敗は**必ず両フィールドに同一メッセージ**を表示します。

## エラー表示のタイミング設計

```
入力開始前  → バリデーションしない（まだ入力途中の可能性）
blur時      → フォーマットエラーのみ表示（必須エラーは出さない）
submit時    → 全エラーを表示
API返却後   → サーバーサイドエラーを表示
```

**blurで必須エラーを出さない理由：** Tabキーでフィールドを移動しただけでエラーが出るのはUX的にストレスだから。

## アクセシビリティ対応
base uiのそのままだとaria-requiredがつかないので注意
またaria-describedbyもつかないので自分で実装する
実装しないとエラーメッセージが読み上げられない
roleに関してはbaseui側がいい感じに定義してくれているのでライブラリ使う場合は要チェック

```tsx
<div>
  <label htmlFor="email">
    メールアドレス
    <span aria-label="必須">（必須）</span>
  </label>
  <input
    id="email"
    type="email"
    aria-describedby="email-error"  // エラーと紐付け
    aria-invalid={!!error}          // スクリーンリーダー用
    aria-required
  />
  {error && (
    <span id="email-error" role="alert">  // role="alert"で即時読み上げ
      {error}
    </span>
  )}
</div>
```

`aria-invalid` と `role="alert"` の2つが最低限必要です。

## SmartHRで特に重視していること

1. **エラーを出す前に防ぐ** — `type="email"` や `maxlength` でそもそも間違えさせない
2. **1フィールド1メッセージ** — 複数エラーがあっても最優先の1つだけ表示する
3. **色だけに依存しない** — 赤色＋アイコン＋テキストの三点セットでエラーを伝える（色覚多様性への配慮）
4. **文体を統一する** — 「〜してください」で統一し、「〜が必要です」「〜は無効です」と混在させない
